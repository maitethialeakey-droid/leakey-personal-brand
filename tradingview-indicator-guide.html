How to Build a TradingView Indicator (Beginner Guide)

Posted on April 13, 2026 | 10 min read | By Leakey Maitethia

Introduction: Why Build Custom Indicators?

TradingView's built-in indicators are powerful, but they're generic. Custom indicators let you implement your exact trading strategy, automate your analysis, and create alerts that match your specific needs. Pine Script makes this accessible even for beginners.

In this guide, you'll learn Pine Script fundamentals and build your first custom indicator from scratch.

What is Pine Script?

Pine Script is TradingView's programming language designed specifically for creating trading indicators, strategies, and alerts. It's beginner-friendly yet powerful enough for professional traders.

Why Pine Script?

  • Easy to learn—simple syntax similar to Python
  • Direct integration with TradingView charts
  • Access to real-time market data
  • Ability to create alerts and automate trading decisions
  • Large community with shared scripts and examples

Pine Script Basics

The Indicator Structure

Every Pine Script indicator follows this basic structure:

//@version=5
indicator("My First Indicator", overlay=true)

// Your code here

plot(close, color=color.blue)

Key Components

  • @version: Specifies which version of Pine Script to use (currently v5)
  • indicator(): Declares this as an indicator and sets its name
  • overlay: If true, plots on the chart; if false, opens a separate window
  • plot(): Draws lines or values on the chart

Building Your First Indicator: Simple Moving Average

Step 1: Create the Basic Structure

//@version=5
indicator("Simple Moving Average", overlay=true)

// Calculate 20-period moving average
ma20 = ta.sma(close, 20)

// Plot the moving average
plot(ma20, color=color.blue, linewidth=2)

Step 2: Add Customizable Parameters

//@version=5
indicator("Customizable Moving Average", overlay=true)

// User inputs
length = input(20, title="MA Length")
ma_color = input(color.blue, title="MA Color")

// Calculate moving average
ma = ta.sma(close, length)

// Plot the moving average
plot(ma, color=ma_color, linewidth=2)

Step 3: Add Multiple Moving Averages

//@version=5
indicator("Dual Moving Average", overlay=true)

// User inputs
fast_length = input(10, title="Fast MA")
slow_length = input(20, title="Slow MA")

// Calculate moving averages
fast_ma = ta.sma(close, fast_length)
slow_ma = ta.sma(close, slow_length)

// Plot moving averages
plot(fast_ma, color=color.green, linewidth=2)
plot(slow_ma, color=color.red, linewidth=2)

Advanced Concepts

Creating Alerts

You can create alerts that trigger when specific conditions are met. For example, alert when fast MA crosses above slow MA:

// Detect crossover
crossover = ta.crossover(fast_ma, slow_ma)

// Create alert
alertcondition(crossover, title="MA Crossover", message="Fast MA crossed above Slow MA!")

Using Conditional Logic

Create more complex indicators using if/else statements:

// Check if price is above moving average
if close > ma20
plot(ma20, color=color.green)
else
plot(ma20, color=color.red)

Common Indicator Patterns

Pattern 1: Overbought/Oversold Levels

Create an indicator that shows when price is overbought or oversold:

  • Calculate RSI (Relative Strength Index)
  • Plot horizontal lines at 70 (overbought) and 30 (oversold)
  • Color the RSI based on these levels

Pattern 2: Support and Resistance

Automatically identify swing highs and lows:

  • Compare current candle to previous candles
  • Mark swing highs and lows with horizontal lines
  • Create alerts when price approaches these levels

Pattern 3: Trend Confirmation

Combine multiple indicators to confirm trends:

  • Use moving averages for trend direction
  • Use RSI for momentum confirmation
  • Use volume for strength confirmation

Best Practices for Pine Script

  • Keep it simple: Start with basic logic, then add complexity
  • Use meaningful names: Make your code readable and maintainable
  • Add comments: Explain what each section does
  • Test thoroughly: Backtest your indicator before using it live
  • Optimize parameters: Find the best settings for your strategy
Pro Tip: Use TradingView's Pine Script reference documentation. It's comprehensive and includes examples for almost every function you'll need.

Common Mistakes to Avoid

  1. Over-optimization: Don't create indicators that only work on historical data
  2. Ignoring repainting: Some indicators repaint (change past values), which is unreliable
  3. Too many indicators: More indicators don't mean better results
  4. Not testing: Always backtest before using live
  5. Ignoring edge cases: Test your indicator in different market conditions

Next Steps

Now that you understand Pine Script basics, you can:

  • Build indicators based on your trading strategy
  • Create automated alerts for your setups
  • Backtest your indicators on historical data
  • Share your indicators with the TradingView community
  • Combine multiple indicators for better accuracy

Conclusion

Building custom TradingView indicators is easier than you think. Start simple, test thoroughly, and gradually add complexity as you become more comfortable with Pine Script. The best traders use custom indicators tailored to their exact strategy.
